Pyarmor

一个能够加密Python源代码的小工具，能够保 护运行时刻的Python二进制代码不被泄露，设置加密后Python源代码的有 效期限，绑定加密后的Python源代码到硬盘、网卡等硬件设备

加密原理 快速开始

无缝替换

加密后的Python源代码也是一个正常的Python脚本。只需要将一个模块pytransform.py，和几个额外的文件，加入到项目里面，就可以把原来的Python脚本全部替换成为加密后的脚本，然后使用和以前一样的方式运行这些加密脚本。

这个特性也使得经过Pyarmor加密后的脚本，还可以在继续使用第三方代码工具，例如 py2exe, py2app, PyInstaller 打包发布

动态加密

Pyarmor不仅仅加密Python源代码，即便在脚本被解释执行的运行过程中，一旦函数（代码块）执行完成，就被重新加密，从更深层次保证了Python脚本的安全性

对于使用 Python包（Package）进行发布的代码，这种方式更能够提供有效的安全保障

加密脚本的许可方式

设置使用期限

Pyarmor 可以为加密的脚本设置使用期限，过期之后加密的Python脚本就无法在继续运行

绑定加密的Python脚本到固定机器

Pyarmor 可以绑定加密后的Python脚本到硬盘序列号、网卡的Mac地址、IP地址等

Pyarmor的商业客户

快速开始

安装 Pyarmor

最简单的方式是 pip, 或者从 PyPi 上面下载 pypi/pyarmor

pip install pyarmor
加密脚本

使用 pyarmor.py，只需要一 个 obfuscate 命令，就可以递归加 密 --src 目录下面的所有 .py 文 件

cd /path/to/pyaror
python pyarmor.py obfuscate --src=examples/simple --entry=queens.py
运行加密脚本

加密后的脚本和运行加密脚本需要的额外文件都被生成在默认目录 dist

切换到输出目录，像运行正常角本一样运行加密后的脚本 queens.py

cd ./dist
python queens.py
生成不同许可文件

通常需要先得到目标机器的硬件设备信息，Pyarmor 提供了一个命令 hdinfo，可以显示当前机器的硬件设备信息

然后就可以使用另外一个命令 licenses, 创建新的许可文件。新许可文件会存放在当前目录下 licenses/CODE/license.lic，这里CODE就是customer-jondy

接着替换到默认的许可文件 dist/license.lic，默认许可文件允许加密后的脚本运行在任何电脑上，并且永不过期

最后，在新的许可证下再次运行加密脚本 queens.py 

python pyarmor.py hdinfo
python pyarmor.py licenses \
                --expired "2018-12-31" \
                --bind-disk "100304PBN2081SF3NJ5T" \
                --bind-mac "70:f1:a1:23:f0:94" \
                --bind-ipv4 "202.10.2.52" \
                customer-jondy

cp licenses/customer-jondy/license.lic ./dist

cd dist
python queens.py
WebUI

Pyarmor 还提供了一个网页版的图形界面，使用起来比较方便，但是只包含了最亮眼的功能，高级功能还是要用命令行

运行 pyarmor-webui 会首先启动了一个本地 Web 服务，然后自动在浏览器中打开 Pyarmor 的网页版

pyarmor-webui
